# Beitragen

Wir freuen uns über Beiträge zum Projekt!

## Entwicklungsprozess

1. Forken Sie das Repository.
2. Erstellen Sie einen neuen Branch für Ihre Änderungen.
3. Implementieren Sie Ihre Änderungen und fügen Sie Tests hinzu.
4. Stellen Sie sicher, dass alle Tests bestehen.
5. Erstellen Sie einen Pull Request.

## Code-Stil

Wir verwenden:
- **Black** für die Formatierung.
- **Ruff** für das Linting.
- **mypy** für die Typprüfung.

Führen Sie `pre-commit install` aus, um diese Prüfungen automatisch vor jedem Commit durchzuführen.
